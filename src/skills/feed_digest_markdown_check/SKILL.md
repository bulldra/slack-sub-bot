以下の記事のMarkdown形式を検証し、修正した記事全文のみを出力せよ。説明やコメントは不要。

## チェック項目

1. URLのリンク形式: すべてのURLがMarkdownリンク形式になっていること。裸のURL（ `https://...` がリンク記法に包まれていない状態）は禁止。裸URLが見つかった場合、URLの内容から適切なタイトルを推測してリンク形式に変換する
   - X（Twitter）のポストURL（`https://x.com/...` や `https://twitter.com/...`）の場合、リンクテキストは `{user_id} - x.com` の形式にする。例: `[bulldra - x.com](https://x.com/bulldra/status/123456)`
2. 見出し構造: タイトルは `# ` (H1)、セクション見出しは `## ` (H2) のみ。`### ` (H3) 以下は使用しない
3. 禁止記法の除去: 太字（`**text**`）、絵文字、箇条書き（`- ` / `* `）、番号リスト（`1. `）、引用ブロック（`> `）、ダッシュ記号（——）があれば除去または文章に変換する
4. 段落構成: 1文だけの短い段落が3つ以上連続していないこと

## 修正方針

- 記事の内容・論旨は一切変更しない
- 形式の修正のみ行う
- 修正不要な場合はそのまま出力する
